### 2. MCP工具使用规范

#### context7文档搜索工具
- **使用场景：**
  - 需要查阅技术文档、API参考、框架文档时
  - 解决特定技术问题需要官方文档支持
  - 验证最佳实践和设计模式的应用
  - 查找编程语言特性或库的使用方法

- **使用原则：**
  - 优先搜索官方文档和权威来源
  - 明确搜索关键词，避免模糊查询
  - 对搜索结果进行技术验证和交叉验证
  - 引用文档时注明来源和版本

#### Playwright浏览器自动化工具
- **使用场景：**
  - 需要验证前端页面渲染效果、交互行为时
  - UI/UX 走查，检查页面布局、样式、响应式表现
  - 表单填写、按钮点击等用户操作流程验证
  - 截图对比，确认修改前后的视觉差异
  - 调试前端接口请求、控制台报错

- **使用原则：**
  - **分辨率规范（必须严格遵守）：**
    - 移动端（uniapp H5）：使用 iPhone 14 Pro Max 分辨率 `430×932`
    - 后台管理端（admin）：使用默认桌面分辨率 `1280×800`
    - PC前台（pc）：使用默认桌面分辨率 `1280×800`
  - **操作前必须设置分辨率：** 每次打开新页面或切换端时，先调用 `browser_resize` 设置对应分辨率
  - **优先使用 snapshot 而非 screenshot：** `browser_snapshot` 返回可交互的无障碍树，用于定位元素和执行操作；`browser_take_screenshot` 仅用于视觉确认
  - **操作流程：** snapshot 定位元素 → 执行操作（click/type/fill_form）→ snapshot 验证结果
  - **等待策略：** 页面导航或异步操作后，使用 `browser_wait_for` 等待关键文本出现，避免操作未加载完成的页面
  - **网络请求检查：** 使用 `browser_network_requests` 排查接口调用问题，关注失败请求和状态码
  - **控制台监控：** 使用 `browser_console_messages` 检查前端报错，level 设为 `error` 快速定位问题
  - **截图存放路径：** 所有截图必须保存到 `screenshots/` 目录下
  - **截图命名规范：** 按 `screenshots/{端}-{页面}-{状态}.png` 命名，如 `screenshots/admin-login-success.png`、`screenshots/mobile-home-loaded.png`
  - **表单操作：** 优先使用 `browser_fill_form` 批量填写，避免逐个字段 type
  - **标签页管理：** 多页面对比时使用 `browser_tabs` 管理，操作完成后关闭多余标签页

- **移动端专项规范：**
  - 打开 uniapp H5 页面前，必须先 `browser_resize` 为 `430×932`
  - 关注触摸区域大小（最小 44×44px）
  - 验证底部导航栏不被内容遮挡
  - 检查长列表滚动性能和加载状态

- **后台管理端专项规范：**
  - 打开 admin 页面前，确认分辨率为 `1280×800`
  - 验证侧边栏折叠/展开状态下的布局
  - 检查表格在数据量大时的分页和滚动表现
  - 验证弹窗（Dialog）居中显示且不溢出

### 4. 编程原则执行

**每次代码变更都要体现：**

**KISS (简单至上)：**
- 追求代码和设计的极致简洁
- 拒绝不必要的复杂性
- 优先选择最直观的解决方案

**YAGNI (精益求精)：**
- 仅实现当前明确所需的功能
- 抵制过度设计和未来特性预留
- 删除未使用的代码和依赖

**DRY (杜绝重复)：**
- 自动识别重复代码模式
- 主动建议抽象和复用
- 统一相似功能的实现方式

**SOLID原则：**
- **S：** 确保单一职责，拆分过大的组件
- **O：** 设计可扩展接口，避免修改现有代码
- **L：** 保证子类型可替换父类型
- **I：** 接口专一，避免"胖接口"
- **D：** 依赖抽象而非具体实现
